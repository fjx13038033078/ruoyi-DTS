# 音乐剧票务管理系统 - 功能说明文档

## 一、系统概述

本系统基于若依（RuoYi）框架开发，实现演出票务的完整业务流程，包括演出管理、场次管理、座位配置、订单管理、积分兑换等核心功能，并内置简单的防黄牛策略。

---

## 二、核心功能与实现方式

### 2.1 演出管理

**功能描述**：管理演出/剧目基本信息，包括标题、海报、详情、推荐状态、上架状态等。

**实现方式**：
- **数据表**：`biz_performance`
- **后端**：`BizPerformance` 实体 → `BizPerformanceMapper` → `IBizPerformanceService`
- **前端**：`ruoyi-ui/src/views/ticket/performance/index.vue`
- **图片处理**：海报使用若依封装的 `ImageUpload`（上传）、`ImagePreview`（预览）组件
- **接口**：`/ticket/performance/*`（GET 列表/详情，POST 新增/修改/删除/导出）

---

### 2.2 场次管理

**功能描述**：管理每场演出的具体场次，包括场馆、开始时间、售票时间、状态；支持批量生成座位。

**实现方式**：
- **数据表**：`biz_performance_session`
- **后端**：`BizPerformanceSession` 实体 → `BizPerformanceSessionMapper` → `IBizPerformanceSessionService`
- **前端**：`ruoyi-ui/src/views/ticket/session/index.vue`
- **座位配置弹窗**：通过 `batchGenerateSeats` 接口，传入场次ID、区域名称、票价、排数、每排座位数，按「X排Y座」格式批量生成座位
- **接口**：`/ticket/session/*`，`/ticket/session/batchGenerateSeats`（批量生成座位）

---

### 2.3 座位与票价

**功能描述**：管理场次下的座位，支持不同区域不同票价；支持 15 分钟锁座机制。

**实现方式**：
- **数据表**：`biz_session_seat`（含 `status`：0待售/1锁定中/2已售出，`lock_expire_time` 锁定过期时间）
- **后端**：`BizSessionSeat` 实体 → `BizSessionSeatMapper` → `IBizSessionSeatService`
- **锁座机制**：选座下单时调用 `lockSeatsByIds` 锁定座位并设置过期时间；定时任务 `SeatLockTask` 扫描过期锁座，释放座位并取消对应订单
- **接口**：`/ticket/seat/*`

---

### 2.4 订单管理

**功能描述**：管理购票订单，记录用户、场次、金额、支付状态、交易流水等。

**实现方式**：
- **数据表**：`biz_order`（主表）、`biz_order_ticket`（明细表，记录每张票对应的座位）
- **后端**：`BizOrder` / `BizOrderTicket` 实体 → 对应 Mapper → `IBizOrderService` / `IBizOrderTicketService`
- **选座下单**：`IOrderPlacementService.placeOrder(userId, sessionId, seatIds)` 完成选座、锁座、创建订单及明细
- **接口**：`/ticket/order/*`

---

### 2.5 积分兑换

**功能描述**：管理积分商品及兑换记录；用户积分存储在 `sys_user.points`。

**积分累计规则**：支付成功后，按支付金额（元）增加积分，1 元 = 1 积分，四舍五入取整。在 `AlipayController.alipayReturn()` 中，订单状态更新为已支付后自动调用 `sysUserService.addPoints()`。

**实现方式**：
- **数据表**：`biz_points_item`（商品）、`biz_points_record`（兑换记录）
- **后端**：`BizPointsItem` / `BizPointsRecord` 实体 → 对应 Mapper → `IBizPointsItemService` / `IBizPointsRecordService`
- **用户积分**：`SysUser` 实体新增 `points` 字段，用于存储观众可用积分
- **积分增加**：`ISysUserService.addPoints()`、`SysUserMapper.addPoints`
- **接口**：`/ticket/pointsItem/*`（商品管理）

---

### 2.6 剧目收藏

**功能描述**：用户收藏演出剧目。

**实现方式**：
- **数据表**：`biz_user_favorite`（唯一约束：user_id + performance_id）
- **后端**：`BizUserFavorite` 实体 → `BizUserFavoriteMapper` → `IBizUserFavoriteService`

---

## 三、防黄牛策略

### 3.1 策略说明

采用**双重限制**的简单防黄牛策略：

| 策略项 | 限制值 | 说明 |
|--------|--------|------|
| 单笔订单最大购票数 | 4 张 | 限制单次囤票量 |
| 每人每场最大购票数 | 4 张 | 限制同一用户在同一场次的累计购票量 |

### 3.2 实现方式

- **配置**：`TicketConstants` 中定义 `MAX_TICKETS_PER_ORDER`、`MAX_TICKETS_PER_USER_PER_SESSION`
- **校验时机**：
  1. **选座下单**：`OrderPlacementServiceImpl.placeOrder()` 在创建订单前校验
  2. **单张加票**：`BizOrderTicketServiceImpl.insertBizOrderTicket()` 在插入前调用 `IAntiScalperService.validateBeforeAddTicket()`
- **统计逻辑**：`countUserValidTicketsForSession` 统计用户在某场次下「待支付 + 已支付」订单中的有效票数量
- **异常**：违反规则时抛出 `ServiceException`，提示具体限制信息

### 3.3 扩展建议

- 将限制值改为从 `sys_config` 或配置中心读取，便于运营调整
- 可增加实名认证、账号等级、购买冷却时间等策略

## 四、技术架构

### 4.1 后端

- **框架**：Spring Boot + MyBatis
- **模块**：`ruoyi-ticket`（票务业务）、`ruoyi-admin`（管理端接口）
- **包结构**：`com.ruoyi.ticket.domain`、`mapper`、`service`、`constant`
- **接口风格**：统一使用 GET/POST，返回 `AjaxResult`，分页使用 `startPage()` + `getDataTable()`
- **权限**：`@PreAuthorize` 控制接口权限（如 `ticket:performance:list`）

### 4.2 前端

- **框架**：Vue2 + Element UI
- **目录**：`ruoyi-ui/src/views/ticket/`、`ruoyi-ui/src/api/ticket/`
- **组件**：`ImageUpload`、`ImagePreview`、`Pagination`、`RightToolbar` 等若依封装组件
- **权限**：`v-hasPermi` 控制按钮显示

### 4.3 数据库

- **表前缀**：`biz_`（业务表）
- **关键表**：`biz_performance`、`biz_performance_session`、`biz_session_seat`、`biz_order`、`biz_order_ticket`、`biz_user_favorite`、`biz_points_item`、`biz_points_record`
- **扩展**：`sys_user` 新增 `points` 字段

## 七、附录：数据表关系

```
biz_performance (演出)
    └── biz_performance_session (场次)
            └── biz_session_seat (座位)
                    └── biz_order_ticket (订单明细)
                            └── biz_order (订单)
                                    └── sys_user (用户)

biz_user_favorite (收藏) → biz_performance, sys_user
biz_points_item (积分商品) ← biz_points_record (兑换记录) → sys_user
```
